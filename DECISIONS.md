| D1 | Repo root **is** the monorepo root (`pipeline/`, `api/`, `viewer/`, `docs/`) | Brief sketches a `sceneforge/` folder; an extra nesting level inside an already-dedicated repo adds nothing. |
| D2 | No `STANDARD.md` / Drawing House LLD standard exists in the workspace (checked first) → one-page equivalent at `docs/LLD.md` with the seven required sections | Brief §3 fallback path. |
| D3 | Geometry default: COLMAP features/matching + **global SfM**, auto-detecting `colmap global_mapper` (COLMAP ≥ 4.1) with fallback to the legacy standalone `glomap` binary | Verified 2026-07-17: GLOMAP repo archived 2026-03-09, merged into COLMAP 4.1.0. | D4 | Splat training implemented directly on gsplat's `rasterization` + **MCMCStrategy** (not nerfstudio/splatfacto) | MCMC's `cap_max` gives a hard gaussian-count ceiling → the ≤25 MB payload budget is enforceable, not aspirational. | D5 | Splat loss = L1, + DSSIM only if `fused_ssim` happens to be installed | Keeps the mandatory dependency set permissive & small; quality delta is acceptable for MVP and measured in QUALITY.md when GPU runs happen. Revisit after Phase-1 acceptance. |
| D6 | Compressed delivery format: `.ksplat` (GaussianSplats3D converter, MIT), fallback to `.splat` (32 B/gaussian) written by our own numpy writer when node is unavailable | Verified: gsplat's PNG "SOGS-like" compression is NOT the PlayCanvas SOGS container, and SPZ python tooling is thinner; .ksplat is native to the chosen viewer. Converter round-trip is integration-tested against the real loader. |
| D7 | COLMAP binary model I/O implemented in ~200 lines of pure Python instead of depending on pycolmap | CPU tests (CI) stay wheel-free; format is stable and documented (colmap.github.io/format.html); pycolmap remains available in the worker image if ever needed. |
| D8 | Scale fallback chain: door prior → user dimension → **camera-height prior (~1.4 m, confidence 0.3)** → none; `camera_height_prior` added to the schema enum | Without any scale the 1.0–1.5 m wall slice is meaningless. Presenting camera-prior-scaled output while claiming `none` would be dishonest; naming the method in the contract is more transparent. Warning always emitted. |
| D9 | Floor plan SVG is hand-generated (no svgwrite) | svgwrite archived 2024-08-19 (verified). ~150 lines of trivially auditable string building; renders only from the validated contract document. |
| D10 | Queue: arq on **Valkey** (BSD-3), not Redis 8 | Redis 8 is RSALv2/SSPLv1/AGPLv3; Valkey keeps the always-on stack fully permissive. arq is MIT but maintenance-mode → RQ (BSD-2) is the named fallback; the `JobQueue` interface isolates the swap. |
| D11 | Worker contract: one `execute_build(BuildJob) → BuildOutcome` runs on the GPU host; adapters (local/modal/hf_space) only choose WHERE it executes; artifacts move via presigned URLs in both directions | Dispatcher never proxies bytes; GPU provider swap is config-only (brief §5 architecture rule). |
| D12 | Worker↔API auth reuses customer API-key machinery (a dedicated key minted for the worker) rather than a second auth scheme | Less surface, same hashing path; `_result` endpoint is excluded from the public OpenAPI schema. |
| D13 | Tenant privacy: cross-tenant scene access returns 404 (not 403) | Don't leak scene-id existence across API keys. |
| D14 | Viewer pins three@0.170.0 with GaussianSplats3D 0.4.7 (peer `three>=0.160`) | 0.4.7 (2025-01) was developed against the 0.16x–0.17x line; three 0.185 is untested with it. GaussianSplats3D is in maintenance mode — Spark (World Labs) is the successor candidate, license to verify before any swap. |
| D15 | `viewer/dist/rf.js` is committed | The single-file bundle IS the deliverable; serving from GitHub Pages/CDN requires no build infra (zero-cost constraint). |
| D16 | CLI `viewer.html` loads three/GaussianSplats3D from jsDelivr (pinned) rather than inlining ~700 KB into every scene output | Self-contained *folder*, instant local inspection; production embeds use rf.js. Trade-off: needs internet on first view. |
| D17 | OCI Always-Free sizing updated: **2 OCPU / 12 GB** Ampere A1 (halved 2026-06-15, verified) — compose stack sized accordingly | The brief's 4 OCPU/24 GB figure is stale. api+postgres+valkey fit comfortably in 12 GB. |

| D23 | Review fix: gsplat MCMC loop reordered to backward → optimizer.step → `step_post_backward(lr=current_decayed_lr)` | Verified against gsplat v1.5.3 `examples/simple_trainer.py` + `strategy/mcmc.py` source (fetched, not remembered); my first draft called the strategy before the optimizer step and passed the initial lr. |
